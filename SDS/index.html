Introduction

1.1 Purpose


This Software Design Specification document aims to represent the detailed design of our project named “TeamTrack” where people can create To-do lists for a project/work. They can work as a team. The users can join, create a group and can set out to make their to-do lists.It is written based on the Software requirement specification(SRS) of the teamTrack project.
The following link directs to the SRS:
https://bitbucket.org/nabeel_mohammed/nsu.nbm.summer2019.cse327.1.t11/src/master/SRS/index.html
This document will contain:
1.Architectural Pattern used in this project.
2.Class Diagrams
3.Detailed description of each class
4.Database Design
5.Database details
6.Services Used in the project


1.2 Scope of this Project


This project will enable user to set up their goals of a project by creating distinct cards/jobs for their individual work. They can work together as a team or can work alone. Also, they can set deadline of their works. It can help people to develop a project related stuff in a disciplined manner.

2.3 Design


2.4 Design Pattern


For this project we have selected MVC framework which is Model-View-Controller framework. The reason behind picking this one is, MVC architecture allows application logic to be separated from user interface.
a. Model controls the data flow and the behaviour of the system. It handles request for the information and respond to it.
b. View works with the user interface.
c. After receiving the user input from the controller it makes call to the model object.

3.2 Class Details


Member: This class will contain all the necessary information about an user after the registration and the stored data will be cross-checked during log in. The attributes of this class is: memberID, memberName, email, password, RegistrationDate, login(), logout().

Team Member: Team member can create a team or can act alone to create a story. Team can give update about the userStory as well as change the userStory details. The attributes of this class is: memberName(from Member Class), memberID(from Member Class),teamName,team ID, ,updateProgress().

Team Leader: It defines a leader who can create the team. The attributes of this class is: TeamLeaderID, memberID, memberName, teamID, teamName, userStoryID, userStoryName ,createTeam(), addMember(),CreateUserStory(),assignUserStory().

Team: team members work as a team to create/update user story. The attributes of this class is: memberID, memberName, teamID, teamName.

UserStory: It manages the tasks assigned by the team to respective team members. The attributes of this class is: teamID(from team class),userStoryID(), userStoryTitle, description, deadline, UserStoryCreatedTime, createStory(), createdBy(), AssignedTo().

Update: team members can update user story informations if needed. The attributes of this class is: TeamID(from team class), UserStoryName,userStoryID, UserStoryCreatedTime, description(from userStory class).

Admin: Admin looks after the system.
